# Clarify Session 2026-04-04 — Issue #179 銷售方案商品數量自訂

## Idea

銷售方案中的每個商品可以自由設定數量（目前固定為 1），讓管理員能建立如「2 門課程 + 3 件 T-shirt」的組合方案。

## Q&A

### Q1: 數量的資料儲存格式？
- 選項：(A) 新增獨立 meta key `pbp_product_quantities` JSON (B) 改 `pbp_product_ids` 格式 (C) 完全重構為 JSON
- **選擇 A**：新增 `pbp_product_quantities` postmeta（JSON: `{"商品ID": 數量}`），`pbp_product_ids` 維持不動，向下相容風險最低

### Q2: 原價參考值是否乘以數量？
- 選項：(A) 原價 = 各商品 (單價 x 數量) 加總 (B) 原價 = 各商品單價加總（不管數量）
- **選擇 A**：乘以數量才反映真實方案原價

### Q3: 前台數量為 1 時的顯示方式？
- 選項：(A) qty > 1 才顯示「x3」 (B) 所有商品都顯示數量
- **選擇 A**：qty=1 不顯示，保持簡潔

### Q4: 當前課程的處理方式？
- 原選項：(A) 獨立 meta key (B) 放入 pbp_product_ids (C) 統一用 pbp_product_quantities
- **用戶自訂回覆**：當前課程重構為與添加商品的邏輯和 UI 一樣。建立銷售方案時預設添加「當前課程」x1，且可以移除。廢棄 `exclude_main_course` 開關。

### Q5: 數量輸入框的上限值？
- 選項：(A) 上限 99 (B) 上限 999 (C) 不設上限
- **選擇 B**：上限 999

### Q6: 訂單中如何顯示數量？
- 選項：(A) WC qty 欄位顯示實際數量 (B) 數量寫在商品名稱中，WC qty 為購買份數
- **選擇 B**：「超值學習包 - Python 講義 x3」x 1，代表 1 份方案包含 3 本講義

### Q7: 購買 2 份方案（含講義 x3），庫存減多少？
- 選項：(A) 庫存減 6 (B) 庫存減 3
- **選擇 A**：2 x 3 = 6，完全按數學計算

## 決策摘要

| 編號 | 決策 | 選擇 |
|------|------|------|
| Q1 | 儲存格式 | A — 新增 `pbp_product_quantities` JSON meta |
| Q2 | 原價計算 | A — 單價 x 數量 |
| Q3 | 前台顯示 | A — qty > 1 才顯示 |
| Q4 | 當前課程 | 自訂 — 統一為普通商品，自動加入可移除 |
| Q5 | 數量上限 | B — 999 |
| Q6 | 訂單顯示 | B — 數量寫入名稱 |
| Q7 | 庫存扣減 | A — 購買份數 x 方案數量 |
