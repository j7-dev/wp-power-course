# Issue #185 需求澄清紀錄：銷售方案商品數量自由設定

> 日期：2026-04-05 | 提問輪次：1（共 7 題，全部已回答）

---

## 需求摘要

銷售方案中的每個商品可以自由設定數量（含當前課程），結帳後依設定數量扣減庫存。同時移除「排除當前課程」開關，改以統一的商品列表管理。

---

## 澄清結果

### Q1: 數量儲存策略 → **A（平行 meta key）**

- 新增 `pbp_product_quantities` postmeta key
- 值為 JSON 物件：`{"product_id": qty, "product_id": qty}`
- `pbp_product_ids` 結構（多筆 meta row）維持不變
- 缺少 `pbp_product_quantities` 的舊資料，runtime fallback 每個商品數量為 1

### Q2: 當前課程數量意義 → **A（僅影響庫存扣減）**

- 數量僅影響庫存扣減，不影響課程存取權
- 買 1 份方案 → 該課程庫存扣 N（N 為設定的數量），學員仍只獲得 1 個課程存取權

### Q3: 數量上限 → **B（999）**

- 最小值：1
- 最大值：999
- 輸入 0 或空值時自動修正為 1

### Q4: 重複商品 → **A（不允許重複）**

- 同一商品在銷售方案中只能出現一次
- 用數量欄位控制件數
- 嘗試加入已存在的商品時，提示「該商品已在方案中，請直接修改數量」

### Q5: 前台銷售卡片顯示格式 → **B（一律顯示 ×N）**

- 商品名稱後方加 `×數量`
- 數量為 1 時**也顯示** `×1`，格式統一

### Q6: 訂單 bundled item 數量 → **A（最終數量 = 方案數量 × 購買份數）**

- bundled item 加入訂單時，`quantity` = 方案內設定數量 × 客戶購買份數
- 例：方案設定 T-shirt ×3，客戶買 2 份 → 訂單中 T-shirt 數量為 6

### Q7: 移除「排除當前課程」功能 → **自訂方案**

**取代方案：**
- 移除 `exclude_main_course` UI 開關
- 當前課程作為**普通商品**出現在銷售方案商品列表中
- 新建銷售方案時，**預設自動加入「當前課程」×1**
- 用戶可自行移除「當前課程」，交互與移除其他商品完全一致
- 用戶可透過搜尋重新加入「當前課程」，交互與加入其他商品一致

**向下相容規則：**

| 舊資料狀態 | 處理方式 |
|---|---|
| `exclude_main_course = 'yes'` | 銷售方案商品維持不變（不含當前課程） |
| `exclude_main_course = 'no'` 或空值 | 銷售方案商品需額外加上「當前課程」×1（Runtime 補入，若 `pbp_product_ids` 尚未包含 course_id） |
