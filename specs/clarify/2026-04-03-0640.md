# 需求澄清紀錄 — 課程線性觀看功能

**Issue**: #170
**日期**: 2026-04-03
**參與者**: j7-dev (需求方), claude (顧問)

## 澄清問題與答案

### Q1. 有子章節的「父章節」本身，是否計入線性鎖定順序？
- **決定**: A — 父章節也是一個學習單元，必須標示完成才能解鎖下一個章節
- **理由**: 維持一致性，扁平順序中每個項目都計入

### Q2. 管理員/教師預覽章節時，是否不受線性觀看限制？
- **決定**: A — 管理員/教師不受限制，可自由存取所有章節
- **理由**: 管理員需要自由檢視任何章節的內容

### Q3. 學員透過 URL 直接存取被鎖定的章節時，後端應如何處理？
- **決定**: A — 重定向到「第一個尚未完成的章節」
- **理由**: 最符合用戶預期，直接帶到正確的位置減少困惑

### Q4. 「前往下一章節」按鈕在下一章被鎖定時應如何表現？
- **決定**: A — 按鈕變為 disabled 狀態並顯示 tooltip（「請先完成本章節」），影片結束也不自動跳轉
- **理由**: 保持按鈕可見讓學員知道還有下一章，但明確告知需先完成當前章節

### Q5. 鎖定規則：是否需要「所有」前面的章節都完成，還是只看「前一個」？
- **決定**: A — 嚴格規則：章節可存取 = 「已完成」或「前面所有章節都已完成」
- **理由**: 嚴格規則才能真正確保學員不會跳過任何章節

### Q6. 課程銷售頁的章節列表是否也要顯示鎖定圖示？
- **決定**: A — 銷售頁不顯示鎖定資訊，維持現有行為
- **理由**: 銷售頁的目的是促進購買，不宜顯示限制性資訊

## 關鍵設計決策摘要

| 項目 | 決定 |
|------|------|
| 資料儲存 | 課程 postmeta `sequential_mode` (yes/no)，預設 no |
| 鎖定規則 | 嚴格連鎖：所有前序章節都完成才解鎖 |
| 章節順序 | 使用 `get_flatten_post_ids()` 扁平順序，父章節也計入 |
| 管理員 | `manage_woocommerce` 權限者不受限制 |
| 後端阻擋 | 鎖定章節 URL 重定向到第一個未完成章節 |
| 前端即時 | toggle-finish API 回傳 unlocked/locked_chapter_ids |
| 下一章按鈕 | 鎖定時 disabled + tooltip，影片結束不自動跳轉 |
| 銷售頁 | 不受影響，不顯示鎖定狀態 |
| 取消完成 | 已完成的後續章節不受影響，未完成的重新鎖定 |
