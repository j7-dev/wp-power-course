## 初步調研結論（無需使用者確認）

- 前端 `CoursesEdit` (`js/src/pages/admin/Courses/Edit/index.tsx`) 以 Refine `useForm` + `antd-toolkit/refine` 的 `dataProvider` 提交。
- 表單流程：`handleOnFinish` → `formatDateRangeData(values, 'sale_date_range', ['date_on_sale_from', 'date_on_sale_to'])` → `onFinish({ ... })`。
- 當 RangePicker 清空時，`sale_date_range` 為 undefined/null，`formatDateRangeData` 的產出是否帶有 `date_on_sale_from`/`date_on_sale_to`（或帶 undefined）是懷疑點。
- `antd-toolkit` 的 dataProvider 以 axios 傳送 body；若內部走 `toFormData`，`undefined` 會被跳過，導致後端收不到該 key，`separator` 拆出來的 `data[]` 也不含該 key，`handle_save_course_data` 完全不會呼叫 `set_sale_price('')`/`set_date_on_sale_from(null)`，因此 DB 保留舊值。
- 後端 `J7\PowerCourse\Api\Course::handle_save_course_data` 只會 `foreach ( $data as $key => $value )` 呼叫 `set_{key}($value)`，**沒有「已知 data fields 全量更新」機制**。
- `Course::format_terms`-adjacent 的 GET response（`inc/classes/Api/Course.php` L402）把 `date_on_sale_from` 以 `(int) timestamp` 回傳，因此若後端 meta 為空字串或 null，`sale_date_range` 會回 `[0, 0]`。前端 RangePicker 在初始化時要能辨識 `[0, 0]` 為「空」而非 1970-01-01。

## Q&A

- Q1 (修復範圍): 盤點並修復「課程編輯頁 (Courses Edit)」所有可清空欄位，建立通用解法；Bundle / Chapter 等其他編輯頁留給後續 issue
- Q2 (清空後 DB 語義): 儲存為空字串 `''`（與 WC 原生 `WC_Product::set_sale_price('')` 行為一致，`_sale_price_dates_from/to` 同樣存 `''`）。**附加要求**：前端 InputNumber、DatePicker.RangePicker 收到空字串/null/undefined 時要 graceful 處理，不能噴 React warning 或 runtime error
- Q3 (前後端合約): 前端在 `handleOnFinish` 顯式把「被清空的欄位」補成空字串（`sale_price: ''`、`date_on_sale_from: ''`、`date_on_sale_to: ''`），後端收到空字串就呼叫對應的 setter 清空
- Q4 (欄位盤點): 僅「可清空的選填欄位」：`sale_price`、`date_on_sale_from`、`date_on_sale_to`、`short_description`、`slug`（可自動重建）、`purchase_note`、`limit_type`/`limit_value`/`limit_unit`、`course_schedule`、`feature_video`、`trial_video`、`button_text`（外部課程選填）、`sku`
- Q5 (date_range 單側清空): 不支援單側清空；後端收到「一側為空另一側有值」時統一視為兩側都清空
- Q7 (前端 graceful 處理): 前後端都修 — 後端 GET response 空值直接回 `null`（不是 `0`、不是 `''`），前端 RangePicker/InputNumber 都加 guard 防禦 `[0, 0]`、`''`、`null`、`undefined` 四種輸入都不會爆掉
- Q8 (測試涵蓋): 雙重保險 — E2E 覆蓋 Q4 清單所有欄位的清空循環（每欄位 1 個 scenario）+ 後端 PHPUnit 覆蓋 Q4 所有欄位（每欄位 1 個 assert）
