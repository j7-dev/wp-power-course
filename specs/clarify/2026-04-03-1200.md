# 2026-04-03 澄清紀錄：Issue #175 課程線性觀看功能

## 澄清問答

### Q1: 直接 URL 存取被鎖定章節的處理方式
**選擇: A — 導向當前應觀看的章節**
自動 redirect 到「第一個解鎖且未完成的章節」，並顯示 toast 提示「請先完成前面的章節才能觀看此內容」。

### Q2: 既有跳躍完成的進度處理（啟用線性觀看後）
**選擇: C — 最遠進度模式**
以最遠已完成章節的排序位置為基準（忽略中間缺口），位置 0 到 (最遠位置+1) 均為解鎖。

### Q3: 鎖定提示文字詳細程度
**選擇: B — 指名提示**
「請先完成『{章節名稱}』才能觀看本章節」— 明確指出當前需完成的章節名稱。

### Q4: 底部導航按鈕與影片自動跳轉
**選擇: B — 顯示但禁用**
「下一個」按鈕灰色不可點擊，影片播完不自動跳轉，改為顯示完成提示。

### Q5: 後端鎖定驗證範圍
**選擇: B — 教室頁面 + toggle API**
頁面渲染時檢查（鎖定則 redirect）+ toggle-finish-chapters API 驗證（不允許完成被鎖定的章節）。

### Q6: 設定存儲方式
**選擇: A — 僅 product meta**
`enable_linear_viewing` 作為 WooCommerce product meta，預設 `'no'`，與現有 `show_course_complete` 等設定模式一致。

### Q7: 管理員預覽模式
**選擇: A — 免除限制**
管理員預覽時所有章節可自由存取，不受線性觀看影響。

## 解鎖演算法摘要

```
輸入: flat_chapters[] (按 menu_order 排序), finished_chapter_ids[]
輸出: unlocked_chapter_ids[], locked_hints{}

1. 若 enable_linear_viewing = 'no' → 全部解鎖
2. 若為管理員預覽 → 全部解鎖
3. 找到 finished_chapter_ids 中 menu_order 最大的章節位置 max_pos
4. 若無完成紀錄 → max_pos = -1
5. 解鎖範圍: 位置 0 到 min(max_pos + 1, last_pos)
6. 鎖定章節的提示: 指向位置 (max_pos + 1) 的章節名稱（即當前需完成的章節）
7. 若 max_pos + 1 已超出最後位置 → 無鎖定章節
```
