## 核心邏輯

### 解鎖判斷演算法

```
function is_chapter_locked(course_id, chapter_id, user_id):
    // 1. 檢查是否開啟線性觀看
    if course.enable_linear_mode != 'yes':
        return false  // 未開啟，不鎖定

    // 2. 管理員/講師豁免
    if user.can('manage_woocommerce') OR user in course.teacher_ids:
        return false

    // 3. 取得展開後的完整排序
    flatten_ids = ChapterUtils::get_flatten_post_ids(course_id)

    // 4. 找到當前章節在排序中的位置
    index = flatten_ids.indexOf(chapter_id)

    // 5. 第一個章節永遠不鎖定
    if index == 0:
        return false

    // 6. 檢查前一個章節是否已完成
    prev_chapter_id = flatten_ids[index - 1]
    return NOT has_finished_at(prev_chapter_id, user_id)
```

### 前端章節列表鎖定圖示

教室側邊欄的章節列表（`classroom/chapters.php` → `get_children_posts_html_uncached`）需要在渲染時：
1. 取得課程的 `enable_linear_mode` 設定
2. 若為 `yes`，計算每個章節的鎖定狀態
3. 被鎖定的章節顯示鎖頭 SVG 圖示
4. 點擊鎖定章節時，JavaScript 攔截跳轉並顯示 Toast

### 資料模型變更

僅新增一個 course product meta：
- Key: `enable_linear_mode`
- Value: `'yes'` | `'no'`
- Default: `'no'`

不需要新增資料表或欄位，完全基於現有的 `finished_at` chaptermeta 判斷。
