# 課程章節線性觀看 — 澄清紀錄

**Issue**: #152
**日期**: 2026-04-01
**參與者**: j7-dev (用戶), claude (AI)

## 決策紀錄

| # | 問題 | 選項 | 決定 | 理由 |
|---|------|------|------|------|
| Q1 | 後端鎖定執行層級 | A: Template+API 都擋 / B: 僅 Template / C: 僅前端 UI | **A** | 安全性考量，防止學員透過 URL 或 API 繞過鎖定 |
| Q2 | 管理員/講師豁免 | A: 全部豁免 / B: 僅管理員 / C: 不豁免 | **A** | 與現有 `is_avl` 權限檢查邏輯一致 |
| Q3 | 管理端 UI 開關位置 | A: 其他設定 tab / B: 章節管理 tab / C: 新建 tab | **A** | 最低成本，與現有 enable_comment 等開關並列 |
| Q4 | 父層章節處理 | A: 只看葉節點 / B: 看有無影片 / C: 所有章節都計入 | **C** | 不管是父層還是子層，都要按順序完成 |
| Q5 | 前端鎖定互動 | A: 鎖頭+可點擊 Toast / B: 鎖頭+disabled / C: 鎖頭+跳轉 | **A** | 可點擊但有提示，比完全 disabled 更友善 |
| Q6 | 章節完成定義 | A: 沿用手動 / B: 增加自動選項 / C: 強制自動 | **A** | 保持簡單，自動完成可後續迭代 |
| Q7 | Meta key 命名 | 用戶指定 | **`enable_linear_mode`** | 值為 `'yes'` / `'no'`，與 `enable_comment` 格式一致，預設 `'no'` |
