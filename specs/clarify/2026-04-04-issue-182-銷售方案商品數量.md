# Issue #182: 銷售方案商品數量自由設定 — 澄清紀錄

**日期**: 2026-04-04
**參與者**: j7-dev (Product Owner), claude (Clarifier Agent)

---

## 澄清問題與決策

### Q1. 數量資料的儲存方式 [工程問題]
- **決策**: A — 新增 `pbp_product_quantities` meta key（JSON 格式 `{"商品ID": 數量}`）
- **理由**: 最小破壞性修改，`pbp_product_ids` 維持現有格式確保向下相容

### Q2. 定期定額（訂閱）商品是否支援自訂數量 [情境問題]
- **決策**: A — 訂閱商品也可以設定數量，行為與一般商品相同
- **理由**: 統一行為最簡單

### Q3. 「排除目前課程」開關的處理 [情境問題]
- **決策**: 自訂方案 — 完全移除 `exclude_main_course` 開關
- **詳細說明**:
  - 建立銷售方案時，「目前課程」×1 為預設商品
  - 目前課程視為一般商品，可修改數量或從方案中移除
  - 遷移策略:
    - `exclude_main_course = 'yes'`: 不修改 `pbp_product_ids`
    - `exclude_main_course = 'no'` 或無此欄位: 加入 course ID 到 `pbp_product_ids`，量為 1
  - 遷移完成後清除 `exclude_main_course` meta

### Q4. 數量欄位的數值上限 [工程問題]
- **決策**: C — 上限 999
- **範圍**: 1–999 正整數

### Q5. 後台數量輸入框的 UI 擺放位置 [情境問題]
- **決策**: A — 在商品類型標籤與刪除按鈕之間加入 InputNumber

### Q6. 前台銷售方案卡片的數量標示格式 [情境問題]
- **決策**: A — 商品名稱後方加上灰色文字 "×N"（數量 > 1 時顯示，= 1 時不顯示）
