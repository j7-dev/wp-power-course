# 澄清紀錄：Issue #147 影片進度自動完成章節

## 日期：2026-03-27 16:00

## 來源
GitHub Issue #147: [需求] 章節影片 vidStack player 進度達 95% 就視為已完成

## 澄清問題與回答

### Q1：跨技術橋接方式（React Player ↔ jQuery finishChapter）
- **選項**：A) Custom DOM Event / B) Player.tsx 直接呼叫 API / C) 共享 Jotai Store
- **回答**：**A — Custom DOM Event**
- **決策**：Player.tsx dispatch `pc:auto-finish-chapter` 事件，finishChapter.ts 監聽並複用現有 AJAX + UI 更新邏輯

### Q2：「自訂嵌入碼」（code 類型）影片的處理
- **選項**：A) 與 YouTube/Vimeo 相同 / B) 不適用 / C) 需要特殊處理
- **回答**：**B — 不適用**
- **決策**：`code` 類型使用獨立 PHP template 渲染，不走 VidStack，自動完成邏輯天然不觸發

### Q3：自動完成 API 呼叫失敗時的行為
- **選項**：A) 靜默失敗 / B) 靜默失敗 + 重試 / C) 顯示低調提示
- **回答**：**A — 靜默失敗**
- **決策**：僅 console.warn 記錄，不影響觀看體驗

### Q4：自動完成與「Ended 倒數跳轉」的時序互動
- **選項**：A) 不處理（fire-and-forget） / B) 等待 API 回應 / C) navigator.sendBeacon
- **回答**：**A — Fire-and-forget**
- **決策**：API 呼叫在 95% 時已發出，伺服器會處理。頁面跳轉不影響

### Q5：手動切換與自動完成 flag 的同步
- **選項**：A) MutationObserver / B) Custom Event 雙向 / C) data attribute / D) finishChapter.ts 統一管理
- **回答**：**D — 統一在 finishChapter.ts 管理**
- **決策**：Player.tsx 只負責 dispatch 事件，finishChapter.ts 管理所有完成狀態 flag

### Q6：極短影片的門檻處理
- **選項**：A) 不限制 / B) 最低時長 / C) 最低已觀看時長
- **回答**：**A — 不限制**
- **決策**：任何長度的影片都適用 95% 規則

### Q7：課程完成（100% 進度）的自動觸發
- **選項**：A) 不需額外處理 / B) 有課程完成頁面需處理 / C) 後端有 action hook
- **回答**：**A — 不需額外處理**
- **決策**：進度條顯示 100% 即可，課程完成相關邏輯由後端 toggle API 內部判斷

## 技術方案摘要

### 架構
```
VidStack onTimeUpdate/onEnded
  → Player.tsx 計算 currentTime/duration（useRef，不觸發 re-render）
  → 首次跨越 95% → dispatch CustomEvent("pc:auto-finish-chapter")
    → finishChapter.ts 監聽
      → 判斷 isFinished + hasAutoFinished
      → 呼叫 toggle API（silent mode: showDialog = false）
      → 更新 UI
```
