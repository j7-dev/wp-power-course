# Clarify Session 2026-04-15 12:00 — Issue #146 續播功能

## Idea

紀錄用戶上次上到哪一個課程，以及幾分幾秒

（PM 延伸需求摘要：自動記錄每個章節播放秒數、下次進入章節自動續播、我的帳戶課程頁顯示「繼續觀看 第 X 章 MM:SS」CTA、課程頁「上次觀看」提示升級、跨裝置同步、已完成章節改為「重看」CTA、到期課程不顯示、不支援影片類型降級為只顯示章節、跳轉失敗靜默降級從 0 播放）

## Q&A

### Round 1（資料模型 / 寫入策略 / 影片來源相容）

- Q1: 進度寫入粒度與「上次觀看課程」指標 → A: **B** — 每 chapter 一筆紀錄，course 層額外維護「最後看的章節 ID」指標（複合策略）
- Q2: 播放秒數寫入頻率 → A: **C** — 10 秒 debounce + 在 `beforeunload` / `visibilitychange:hidden` / `pause` 事件觸發強制 flush
- Q3: 續播時的回退秒數 → A: **B** — 自動回退 5 秒（避免續播在話題中段，保留少量上下文）
- Q4: 進度過短不記錄門檻 → A: **C** — 觀看時長 < 15 秒不寫入續播紀錄
- Q5: 各影片來源 (bunny / youtube / vimeo / code / none) 是否支援秒數續播 → A: **B** — bunny + youtube + vimeo 三者支援秒數續播；code（嵌入碼）/ none 僅記錄「上次觀看的章節」不記秒數

### Round 2（API 介面 / 既有資料相容 / CTA 條件 / 退課邏輯 / 後台顯示）

- Q6: 進度寫入 API 設計 → A: **B** — 擴充現有 `POST /power-course/v2/chapters/{id}/progress`，加入 `last_position_seconds` 欄位（複用既有 endpoint，避免 API 表面增生）
- Q7: 既有用戶歷史進度的相容處理 → A: **A** — 既有用戶無歷史秒數紀錄 → 視為從未播放，從 0 開始（不做歷史回溯）
- Q8: 「繼續觀看」CTA 顯示條件 → A: **B** — 僅在課程「授權中且未到期」時顯示續播 CTA；課程到期 / 未開課 / 未授權則顯示原本 CTA（與既有「課程到期限制計算.feature」對齊）
- Q9: 進度資料的生命週期 → A: **C** — 學員退課（移除課程權限）→ 刪除進度紀錄；課程到期 → 保留紀錄（續訂後可恢復續播）；站長後台手動重設學員進度 → 刪除紀錄
- Q10: 後台「上次觀看」資訊揭露程度 → A: **B** — 站長後台僅顯示「上次觀看章節 + 最後活動時間」，不顯示秒數細節（避免隱私 / 監視疑慮，秒數僅供前端續播使用）
