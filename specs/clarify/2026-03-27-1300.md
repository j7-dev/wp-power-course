# 澄清紀錄：Issue #143 銷售方案商品數量自由填寫

**日期**: 2026-03-27
**來源**: GitHub Issue #143

## 澄清問題與回答

### Q1: 數量輸入的驗證規則
**選項**: A) 最小值 1，無上限 / B) 最小值 1，上限為庫存 / C) 最小值 0，無上限
**回答**: **A** — 最小值 1，無上限

### Q2: 「當前課程」數量 > 1 的業務含義
**選項**: A) 僅影響庫存扣減，授權 1 次 / B) 多席位 / C) 固定 1 不可改 / D) 只影響庫存
**回答**: **A** — 僅影響庫存扣減，課程授權只給 1 次

### Q3: 前台顧客端銷售方案的數量顯示
**選項**: A) 商品名 + 數量 / B) 只顯示商品名 / C) 數量 > 1 才顯示
**回答**: **A** — 顯示商品名稱 + 數量（如「課程 A × 2」）

## 確認的規格摘要

### 資料模型
- 新增 `pbp_product_quantities` meta key（JSON 格式：`{"product_id": quantity}`）
- 與既有 `pbp_product_ids` 並存，向後相容

### 驗證規則
- 數量為正整數，最小值 1，無上限
- 未指定數量時預設為 1

### 訂單處理
- 庫存扣減 = 商品個別數量 × 銷售方案購買數量
- 課程授權：不受數量影響，有買就有 1 次權限

### UI 變更
- 後台：每個商品旁加 InputNumber（min=1, default=1）
- 前台：顯示「商品名 × 數量」格式
