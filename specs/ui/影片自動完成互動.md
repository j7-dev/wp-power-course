# 影片自動完成互動

## 描述
當學員在教室頁面觀看章節影片，播放進度達到 95% 門檻或影片播放結束（ended）時，
前端自動將該章節標記為已完成（靜默模式，不彈出對話框）。

## 架構
- **Player.tsx**（React）：監聽 VidStack 的 `onTimeUpdate` 和 `onEnded` 事件，計算播放百分比
- **finishChapter.ts**（jQuery + Jotai）：監聯 Custom DOM Event `pc:auto-finish-chapter`，執行 API 呼叫和 UI 更新
- **跨框架橋接**：透過 Custom DOM Event（`pc:auto-finish-chapter`）解耦 React 與 jQuery

## 資料流
```
VidStack onTimeUpdate → Player.tsx 計算 currentTime/duration
  → 首次跨越 95% → dispatch CustomEvent("pc:auto-finish-chapter", { chapterId, courseId })
    → finishChapter.ts 監聽事件
      → 檢查 isFinished flag（true 則忽略）
      → 檢查 hasAutoFinished flag（true 則忽略）
      → 呼叫 POST /power-course/toggle-finish-chapters/{chapter_id}
      → 成功：更新 UI（按鈕、Badge、進度條、章節圖示），設定 hasAutoFinished = true
      → 失敗：console.warn，不彈框
```

## 常數
- `FINISH_THRESHOLD = 0.95`：定義在 Player.tsx 頂層，未來可改為從 `window.pc_data` 注入

## 關鍵元素
- VidStack `onTimeUpdate` 事件監聽（計算 `currentTime / duration`）
- VidStack `onEnded` 事件監聯（保底觸發）
- `hasAutoFinished` ref（React useRef）：防止同一次觀看中重複 dispatch
- Custom DOM Event `pc:auto-finish-chapter`：跨框架橋接
- finishChapter.ts 中的 `isFinished` 狀態：判斷章節是否已完成（從 data attribute 初始化）
- finishChapter.ts 中的 `hasAutoFinished` flag：防止重複呼叫 API
- 靜默模式：`showDialog: false`（區別於手動完成的 `showDialog: true`）

## PHP Template 變更
vidstack/index.php 新增 data attributes：
- `data-chapter_id`：當前章節 ID（供 Player.tsx 讀取）
- `data-course_id`：當前課程 ID（供 Player.tsx 讀取）
- `data-is_finished`：當前章節完成狀態（"true" / "false"）

## 效能注意事項
- `onTimeUpdate` 每秒觸發多次，百分比計算使用 useRef 而非 useState，避免不必要的 re-render
- 僅在首次跨越 95% 門檻時 dispatch 事件，後續的 timeUpdate 事件被 ref flag 擋下

## 適用影片類型
| 影片類型 | VidStack 渲染 | 自動完成 |
|---------|:------------:|:-------:|
| bunny (HLS) | Yes | Yes |
| youtube | Yes | Yes（若 VidStack 可追蹤進度） |
| vimeo | Yes | Yes（若 VidStack 可追蹤進度） |
| code | No（獨立 template） | No |
| none | No（無影片） | No |
