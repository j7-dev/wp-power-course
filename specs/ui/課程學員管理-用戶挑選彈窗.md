## 2. UserTable `mode` Prop 規格

```ts
export type UserTableMode = 'global' | 'course-exclude'

export type UserTableProps = {
  /** 模式切換，預設 'global'（維持既有 /admin/students 行為） */
  mode?: UserTableMode
  /** 僅 'global' 模式生效：是否顯示開通/CSV/移除/HistoryDrawer 等管理員功能 */
  canGrantCourseAccess?: boolean
  tableProps?: TableProps<TUserRecord>
  cardProps?: CardProps & { showCard?: boolean }
}
```

### 2.1 模式與行為映射

| 行為 | `mode='global'`（預設） | `mode='course-exclude'` |
|------|------------------------|------------------------|
| Refine resource | `'users'` | `'students'` |
| dataProviderName | 預設 | `'power-course'` |
| permanent filter | `[{ meta_keys: ['is_teacher', 'avl_courses'] }]` | `[{ meta_key: 'avl_course_ids' }, { meta_value ne courseId }, { meta_keys: ['formatted_name'] }]` |
| `queryOptions.enabled` | `true` | `!!courseId`（從 `useParsed().id` 取） |
| 分頁 pageSize | `20` | `20` |
| `Filter` card | 顯示（search + avl_course_ids + include） | **顯示**（三欄位完整保留） |
| `FilterTags` | 顯示 | 顯示 |
| Table columns | 沿用 `useColumns` | 沿用 `useColumns` |
| `rowSelection` | 顯示 | 顯示 |
| `SelectedUser` 已選清單 | 顯示（原由外部調用） | 顯示（由 UserTable 內部 render） |
| `GrantCourseAccess` 按鈕 | `canGrantCourseAccess=true` 時顯示 | **隱藏** |
| 批次「更新觀看到期日」 | `canGrantCourseAccess=true` 時顯示 | **隱藏** |
| `RemoveCourseAccess` | `canGrantCourseAccess=true` 時顯示 | **隱藏** |
| CSV 匯出 + CSV 上傳 Modal | `canGrantCourseAccess=true` 時顯示 | **隱藏** |
| `HistoryDrawer` | 顯示 | **隱藏** |
| **新增**「加入本課程」按鈕 | **不顯示** | **顯示**（模式專屬） |
| **新增** DatePicker（到期日） | **不顯示** | **顯示**（模式專屬） |

## 3. `course-exclude` 模式專屬 UI 細節

### 3.1 加入按鈕（主 CTA）

- **位置**：`Filter` card 之下、`SelectedUser` 之下、Table 之上的 Action Bar 區塊內
- **Label**：
  - 有勾選時：`sprintf(__('Add %s students to this course', 'power-course'), selectedRowKeys.length)`
  - 無勾選時：`__('Add students to this course', 'power-course')`
- **disabled 條件**：`selectedRowKeys.length === 0 || isLoading`
- **樣式**：`type="primary"`
- **loading**：mutation 執行期間 `loading={isLoading}`

### 3.2 DatePicker（到期日）

- **位置**：加入按鈕「右側」，以 `Space.Compact` 或 flex gap-x-4 並排
- **props**：
  ```tsx
  <DatePicker
    value={expireDate}
    onChange={(value: Dayjs | null) => setExpireDate(value ?? undefined)}
    placeholder={__('Leave empty for permanent access', 'power-course')}
    showTime
    format="YYYY-MM-DD HH:mm"
    disabledDate={(current) => current && current < dayjs().startOf('day')}
    disabled={isLoading}
  />
  ```
- **值 → API payload 對照**：
  - `expireDate === undefined` → 送 `expire_date: 0`
  - `expireDate` 是 Dayjs → 送 `expire_date: expireDate.unix()`
- **label hint**（DatePicker 上方小字，可選顯示）：`__('Expire date (applied to all selected)', 'power-course')`

### 3.3 SelectedUser 已選清單

- 沿用 `@/components/user/UserTable/SelectedUser`，在 UserTable 內部 render
- `onClear` 行為：`setSelectedUserIds([])`
- `onSelected` 行為：與既有 UserTable 相同（把已選 ids 填入 Filter `include` 欄位 + form.submit），讓使用者切頁後能快速篩回既有勾選的人

### 3.4 成功/失敗行為

| 結果 | UI 動作 |
|------|---------|
| 成功 | `message.success({ content: __('Students added successfully', 'power-course'), key: 'add-students' })`；`invalidate({ resource: 'students', dataProviderName: 'power-course', invalidates: ['list'] })`；`setSelectedUserIds([])`；`setExpireDate(undefined)` |
| 失敗 | `message.error({ content: __('Failed to add students to course', 'power-course'), key: 'add-students' })`；保留勾選 & DatePicker 不重設 |

> 由於 `selectedUserIdsAtom` 變動時會自動 `setSelectedRowKeys([])`（既有 `useRowSelection` 內的 `useEffect` 邏輯），勾選 UI 的打勾也會跟著清空。

## 4. 互動細節（與 StudentTable 的連動）

- 本 UserTable（`course-exclude`）與下方 `StudentTable` 共用 `resource: 'students'`。加入成功後 `invalidate` 會同時刷新兩者：
  - 上方 UserTable：剛加入的人被 `meta_value ne courseId` 濾掉 → 從列表消失
  - 下方 StudentTable：`meta_value eq courseId` → 剛加入的人出現
- **副作用自動發生**，不需要額外手動 refetch `StudentTable`

## 5. Accessibility / 可用性

- Button `disabled` 時用 `aria-disabled` 自動由 antd 處理
- DatePicker 的 `placeholder` 要能明確傳達「不選 = 永久」，避免管理員誤解為必填
- message.success/error 使用 `key: 'add-students'`，同鍵訊息會互相取代，避免堆疊

## 6. 視覺一致性

- 分頁 `pageSize: 20` → 與 `/admin/students` 全局 UserTable 及下方 `StudentTable` **三者一致**
- 使用 `getDefaultPaginationProps({ label: __('user', 'power-course') })` 保持分頁文案風格統一
- Card 邊距、按鈕 gap、Input placeholder 風格沿用 `UserTable/index.tsx` 既有設計
