# 線性觀看教室互動規格

> Issue #205 | 確認日期 2026-04-20

## 1. 側邊欄章節導航列表

### 1.1 鎖定章節外觀

- 鎖定的章節 `<li>` 新增 CSS class `pc-chapter-locked`
- PHP 渲染時輸出 `data-locked="true"` 和 `data-lock-message="請先完成『{前置章節名稱}』才能觀看本章節"` 屬性
- 鎖定章節的圖示區域（`.pc-chapter-icon`）顯示鎖頭圖示（SVG lock icon）
- 鎖定章節的標題文字降低透明度（`opacity: 0.5`）
- 鎖定章節的提示文字顯示在標題下方，字體較小、灰色

### 1.2 鎖定章節點擊行為

- 學員點擊鎖定章節的 `<li>` 時：
  - **不跳轉頁面**（攔截 `href` 行為）
  - 彈出 HTML `<dialog>` 對話框
  - 對話框標題：「章節尚未解鎖」
  - 對話框訊息：取自 `data-lock-message` 屬性（例如「請先完成『第一單元』才能觀看本章節」）
  - 對話框有「確定」按鈕關閉

### 1.3 即時解鎖更新（DOM 操作）

- 學員完成章節後（toggle-finish API 成功回應），前端：
  1. 從 API 回應取得 `unlocked_chapter_ids` 和 `locked_chapter_ids`
  2. 對 `unlocked_chapter_ids` 中的每個章節：
     - 移除 `pc-chapter-locked` class
     - 移除 `data-locked` 屬性
     - 將鎖頭圖示恢復為原始章節圖示（使用 API 回應的 `icon_html`，或空白圖示）
     - 恢復標題透明度
     - 移除鎖定提示文字
  3. 對 `locked_chapter_ids` 中的每個章節（取消完成時需重新鎖定）：
     - 新增 `pc-chapter-locked` class
     - 設定 `data-locked="true"`
     - 替換圖示為鎖頭
     - 降低標題透明度
     - 插入鎖定提示文字

### 1.4 未啟用線性觀看

- 未啟用時，所有章節 `<li>` 不帶 `data-locked` 屬性
- 外觀與行為與現有系統完全一致

## 2. 底部章節導航按鈕

### 2.1 「下一個」按鈕鎖定狀態

- 下一章被鎖定時：
  - 按鈕仍然顯示，但加上 `disabled` 屬性與 `pc-btn-disabled` class
  - 按鈕呈灰色不可點擊
  - 按鈕文字旁顯示小型鎖頭圖示
  - 按鈕下方或 tooltip 顯示「完成本章節以解鎖下一章」
- 下一章解鎖時：
  - 按鈕恢復正常可點擊狀態
  - 移除 disabled 與灰色樣式

### 2.2 「上一個」按鈕

- 不受線性觀看限制（已完成的章節可自由回顧）
- 維持現有行為

## 3. 影片 Ended 畫面（Ended.tsx）

### 3.1 智慧判斷邏輯（Q4 = C）

```
IF 課程未啟用線性觀看:
  → 現有行為：5 秒倒數 + 自動跳轉
ELSE IF 下一章已解鎖（自動完成已觸發，或先前已完成）:
  → 5 秒倒數 + 自動跳轉（正常行為）
ELSE IF 下一章仍鎖定:
  → 不顯示倒數
  → 顯示引導提示：「請點擊『完成章節』以解鎖下一章」
  → 顯示「完成章節」按鈕（或引導指向底部的完成按鈕）
END
```

### 3.2 判斷依據

- Ended.tsx 需知道「下一章是否鎖定」
- 資料來源：PHP 渲染時透過 `wp_localize_script` 輸出 `pc_data.next_chapter_locked`（boolean）
- 若影片自動完成事件（`pc:auto-finish-chapter`）觸發且 API 成功，前端更新 `next_chapter_locked` 為 `false`

### 3.3 i18n 修正

- 現有硬編碼：`下個章節將在 {countdown} 秒後自動播放`
- 修正為：`sprintf(__('Next chapter will auto-play in %d seconds', 'power-course'), countdown)`

## 4. Toast 提示（URL redirect 後）

### 4.1 觸發條件

- 頁面 URL 包含 `?linear_locked=1` query 參數

### 4.2 行為

1. 頁面載入後，JS 偵測 `URLSearchParams` 中的 `linear_locked` 參數
2. 若為 `1`，在頁面頂部顯示 toast 提示
3. Toast 訊息：「請先完成前面的章節才能觀看該內容」
4. Toast 自動消失（5 秒後），或點擊可關閉
5. 使用 `history.replaceState` 移除 URL 中的 `linear_locked` 參數
6. Toast 樣式：藍色 info 色調，不是紅色 error

## 5. 後台課程設定 UI

### 5.1 開關位置

- 課程編輯頁 →「其他設定」分頁 → 「教室設定」區域
- 開關標籤：「啟用線性觀看（循序學習模式）」
- 使用 Ant Design `Switch` 元件
- 開關下方說明文字：「開啟後，學員必須依序完成章節才能觀看後面的內容」

### 5.2 儲存行為

- 開關切換後，隨課程設定一起透過現有的 course update API 儲存
- 對應 meta key：`enable_linear_viewing`
- 值：`'yes'` / `'no'`

## 6. 章節排序警告 UI

### 6.1 觸發條件

- 管理員在已啟用線性觀看的課程中拖曳調整章節排序

### 6.2 警告方式

- 排序 API 回應成功後，若課程已啟用線性觀看，前端顯示 Ant Design `message.warning`
- 警告文字：「此課程已啟用線性觀看，調整排序將影響學員的章節解鎖狀態」
