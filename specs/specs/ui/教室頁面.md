# 教室頁面

## 描述
學員觀看課程的主要頁面，包含章節列表、影片播放器、進度追蹤。URL slug: `/classroom/{chapter_slug}`。

## 關鍵元素
- 章節導航列表（按 menu_order 排序，標記已完成章節）
- 影片播放器（VidStack，支援 Bunny HLS / YouTube / Vimeo / 自訂嵌入碼）
- 章節內容區域（power-editor / elementor 內容）
- 完成章節按鈕（切換完成/未完成狀態）
- 課程整體進度條
- 留言區塊（若 enable_comment = yes）
- 浮水印覆蓋層（若 pc_watermark_qty > 0）
- 手機端固定影片/Tab（若 fix_video_and_tabs_mobile = yes）

## 線性觀看模式（sequential_mode = yes）

當課程啟用線性觀看模式時，教室頁面有以下額外行為：

### 章節導航列表
- 鎖定的章節顯示**鎖頭圖示**（取代原有的播放/完成圖示）
- 鎖定的章節文字為**灰色**（opacity 降低），與可存取的章節形成視覺區隔
- 學員點擊鎖定的章節時，顯示提示對話框：「請先完成前面的章節才能觀看此章節」，不跳轉頁面
- 標記章節完成後，JS 即時移除下一個解鎖章節的鎖頭圖示（不需重新整理）
- 取消章節完成後，JS 即時為後續未完成的章節重新加上鎖頭圖示

### 前往下一章節按鈕
- 若下一章節處於鎖定狀態，按鈕變為 **disabled** 並顯示 tooltip：「請先完成本章節」
- 標記當前章節完成後，按鈕即時恢復為可點擊狀態

### 影片播放器
- 若下一章節處於鎖定狀態，影片播放結束時**不自動跳轉**到下一章
- 顯示提示：「請先標示本章節為已完成，以解鎖下一章」

### 後端存取控制
- 學員透過 URL 直接存取被鎖定的章節時，後端 **301 重定向**到「第一個尚未完成的章節」
- 管理員（`manage_woocommerce` 權限）不受限制，可自由存取任何章節
