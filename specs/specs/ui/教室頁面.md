# 教室頁面

## 描述
學員觀看課程的主要頁面，包含章節列表、影片播放器、進度追蹤。URL slug: `/classroom/{chapter_slug}`。

## 關鍵元素
- 章節導航列表（按 menu_order 排序，標記已完成章節）
- 影片播放器（VidStack，支援 Bunny HLS / YouTube / Vimeo / 自訂嵌入碼）
- 章節內容區域（power-editor / elementor 內容）
- 完成章節按鈕（切換完成/未完成狀態）
- 課程整體進度條
- 留言區塊（若 enable_comment = yes）
- 浮水印覆蓋層（若 pc_watermark_qty > 0）
- 手機端固定影片/Tab（若 fix_video_and_tabs_mobile = yes）

## 線性觀看模式（enable_sequential = yes）
當課程開啟線性觀看時，教室頁面會有以下額外行為：
- **章節鎖定圖示**：尚未解鎖的章節在側邊欄列表中顯示 🔒 鎖頭圖示，且不可點擊
- **點擊鎖定章節**：顯示友善提示「請先完成前面的章節，才能觀看此章節喔！」
- **完成章節後即時解鎖**：完成章節的 API 回傳 `next_unlocked_chapter_id`，前端 JS 直接移除下一個章節的鎖頭圖示並啟用點擊
- **取消完成確認**：在線性觀看模式下取消完成時，彈出確認對話框「取消完成此章節後，後續章節將重新鎖定，確定要繼續嗎？」
- **直接 URL 存取**：後端驗證存取權限，鎖定章節會被導向到目前應觀看的章節（最後完成章節的下一個）
- **不需要額外提示橫幅**：鎖頭圖示已足夠直覺
