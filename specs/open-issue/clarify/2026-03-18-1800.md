# Clarify Session 2026-03-18 18:00

## Idea

### 需求 1：字幕語言下拉選單預設空選項
- 位置: js/src/components/formItem/VideoInput/SubtitleManager.tsx
- 字幕管理的下拉選單，多一個空的 option，顯示文字為 "請選擇字幕語言"，並且預設選擇這個 option

### 需求 2：字幕上傳功能與章節解耦
- 目前的字幕管理功能似乎是跟章節(chapter)強綁定，沒有章節就會顯示錯誤「章節不存在」
- 但實際上課程(course)詳情頁也會有「課程封面影片」以及「課程免費試看影片」等使用 VideoInput 的狀況，這些場景也有上傳字幕的需求
- 用戶希望字幕是跟著影片的，而不是跟著章節(chapter)

## Q&A

- Q: 字幕存儲方式偏好哪種方案？ → A: 方案 B（以 post ID 為 key，泛化現有方案），但補充約束：一個 post 可能有多個影片（如課程同時有封面影片和試看影片），每支影片各自有獨立的字幕管理。存儲結構需能區分同一 post 下不同影片的字幕。
- Q: 如何區分同一 post 下不同影片的字幕？ → A: 用 video slot 名稱區分。API 路徑為 posts/{id}/subtitles/{videoSlot}/{srclang}，videoSlot 值為 chapter_video / feature_video / trial_video。meta key 為 pc_subtitles_{videoSlot}。
- Q: 後端是否驗證 post type 與 videoSlot 的搭配？ → A: 是。pc_chapter 只允許 chapter_video；product 只允許 feature_video 和 trial_video。不符時回傳 400 錯誤。
- Q: 既有資料如何遷移？ → A: 方案 C，不遷移直接切換。字幕功能用戶尚未開始使用，無既有資料需要遷移。
