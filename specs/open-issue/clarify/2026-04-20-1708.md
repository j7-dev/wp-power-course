# Clarify Session 2026-04-20 17:08 — Issue #206 課程影片觀看完後無法重複觀看

## Idea

"紀錄課程觀看進度 X分X秒" 功能導致的 BUG

如果用戶觀看的進度，是紀錄在接近結束的位置(95% 後)，會進入倒數畫面（Ended 元件覆蓋），此時無法拖拉進度條。

附圖顯示 VidStack 播放器被一層半透明黑色遮罩蓋住，中央有倒數畫面「下個章節將在 X 秒後自動播放」。

### 技術根因（已完成分析）

1. `useChapterProgress.ts` GET `last_position_seconds`（例：580 秒，duration 600 秒，已達 96.7%）
2. `Player.tsx` 在 `onLoadedMetadata` / `onCanPlay` seek 到 580 秒（符合 `續播至上次觀看秒數.feature` Rule「已完成且秒數接近片尾仍正常續播」）
3. 從 580 秒播放約 20 秒自然到 ended → `setIsEnded(true)`
4. `Ended.tsx` 以 `absolute top-0 left-0 w-full h-full bg-black/50 z-10` 覆蓋整個播放器，進度條、控制列全部被遮蔽
5. 5 秒倒數結束 `window.location.href = next_post_url`，學員沒機會重看

### 關聯 feature

- `specs/features/progress/續播至上次觀看秒數.feature` — Rule「已完成且秒數接近片尾仍正常續播」
- `specs/features/progress/影片進度自動完成章節.feature` — Rule「自動完成 API 呼叫為 fire-and-forget，不影響 Ended 倒數跳轉」（本次需調整）
- `specs/features/course/繼續觀看課程CTA.feature` — CTA「重看 第 X 章 MM:SS」語意破產
- `specs/open-issue/004-resume-playback-position.md` — Q12 決策「保留 last_position_seconds；CTA 改為『重看』；仍自動續播」
