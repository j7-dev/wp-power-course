# Power Course i18n（國際化）規範

本規範定義 Power Course 外掛在多語系支援上的強制規則。所有 PHP 與 React 程式碼必須遵守。違規視為 PR 審查不通過。

詳細 API 用法請參考 `wp-workflows:wordpress-i18n` skill。

## 核心決策

| 項目 | 決策 |
|------|------|
| Text Domain | **固定 `power-course`**（連字號，與 `plugin.php` header 一致） |
| Domain Path | `/languages` |
| 翻譯來源 | 單一 `.po/.mo`，PHP 與 React **共用** |
| PHP API | WordPress 原生 `__()` / `_e()` / `_x()` / `_n()` 等 |
| React API | **`@wordpress/i18n`**（已安裝），**禁止**使用 `i18next` / `react-intl` |
| 工具鏈 | `pnpm run i18n:pot`（呼叫 `wp i18n make-pot`）+ `pnpm run i18n:json` |
| 目標語言 | en_US（優先）+ ja（次階段） |

## PHP 規範

### 1. 必須使用翻譯函式

任何**使用者可見的字串**（UI 標籤、訊息、錯誤、按鈕、placeholder、admin 通知、tab 標題）必須包在翻譯函式內，禁止硬編碼。

```php
// ✅ 正確
__( '課程名稱', 'power-course' )
esc_html__( '刪除課程資料失敗', 'power-course' )

// ❌ 錯誤
'課程名稱'  // 硬編碼，無法翻譯
__( '課程名稱' )  // 缺 text domain，make-pot 抓不到
__( '課程名稱', 'power_course' )  // domain 錯：必須是連字號
```

### 2. 強制使用 escape 變體輸出 HTML

直接輸出到 HTML 的字串必須用 escape 版本，**禁止裸用 `_e()` 輸出**：

| 場景 | 必用函式 |
|------|---------|
| 輸出到 HTML body | `esc_html_e()` / `esc_html__()` |
| 輸出到 HTML attribute（`title="..."` / `aria-label="..."`） | `esc_attr_e()` / `esc_attr__()` |
| 輸出到 URL 屬性（`href` / `src`） | `esc_url()` 包外層 |
| 輸出到 JS context | `esc_js()` 包外層 |
| 含 HTML tag 必要的字串 | `wp_kses_post()` 包外層 |

```php
// ✅ 正確
echo esc_html__( '請選擇課程', 'power-course' );
?>
<button title="<?php esc_attr_e( '加入購物車', 'power-course' ); ?>"></button>

// ❌ 錯誤
echo __( '請選擇課程', 'power-course' );  // XSS 風險
_e( '請選擇課程', 'power-course' );  // 同上
```

### 3. text domain 必須是字串字面量

`make-pot` 是**靜態掃描**，禁止把 text domain 放在變數內：

```php
// ✅ 正確
__( '課程名稱', 'power-course' )

// ❌ 錯誤：make-pot 掃不到，永遠無法翻譯
$domain = 'power-course';
__( '課程名稱', $domain )
```

### 4. sprintf + placeholder 規範

帶變數的字串**必須**用 `sprintf` + placeholder，禁止字串拼接：

```php
// ✅ 正確
sprintf(
    /* translators: %s: 課程名稱 */
    esc_html__( '課程「%s」已建立', 'power-course' ),
    esc_html( $course_name )
);

// 多參數時必須用編號 placeholder（譯者可重排序）
sprintf(
    /* translators: 1: 學員姓名, 2: 課程名稱 */
    esc_html__( '%1$s 已加入課程「%2$s」', 'power-course' ),
    esc_html( $student_name ),
    esc_html( $course_name )
);

// ❌ 錯誤：字串拼接
__( '課程「', 'power-course' ) . $course_name . __( '」已建立', 'power-course' )

// ❌ 錯誤：直接代換變數
__( "課程「{$course_name}」已建立", 'power-course' )  // make-pot 掃到的字串永遠變動
```

### 5. 複數規範

數量相關的字串必須用 `_n()`：

```php
// ✅ 正確
sprintf(
    /* translators: %d: 學員人數 */
    _n(
        '%d 位學員',
        '%d 位學員',
        $count,
        'power-course'
    ),
    $count
);

// ❌ 錯誤
sprintf( __( '%d 位學員', 'power-course' ), $count );
```

### 6. translator comments 慣例

含 placeholder 的字串**必須**在前一行加 `/* translators: ... */` 註解，告訴譯者每個 placeholder 的意義：

```php
sprintf(
    /* translators: 1: 折扣百分比, 2: 課程名稱 */
    esc_html__( '享 %1$d%% 折扣購買「%2$s」', 'power-course' ),
    $discount,
    esc_html( $course_name )
);
```

## React / TypeScript 規範

### 1. 必須使用 `@wordpress/i18n`

```tsx
// ✅ 正確
import { __, _x, _n, sprintf } from '@wordpress/i18n';

<Button>{__('儲存課程', 'power-course')}</Button>

// ❌ 錯誤：使用 i18next
import { useTranslation } from 'react-i18next';

// ❌ 錯誤：硬編碼
<Button>儲存課程</Button>
```

### 2. text domain 必須與 PHP 一致

所有 JS 翻譯呼叫的第二參數**必須是 `'power-course'`**（與 PHP 共用）。`wp_set_script_translations` 已在 `inc/classes/Bootstrap.php` 接好，runtime 會自動載入對應翻譯。

### 3. Ant Design 元件文字

Ant Design 的 `label`、`placeholder`、`title`、`tooltip`、`okText`、`cancelText` 等 props 都是使用者可見字串，必須翻譯：

```tsx
// ✅ 正確
<Form.Item
  label={__('課程名稱', 'power-course')}
  name="name"
>
  <Input placeholder={__('請輸入課程名稱', 'power-course')} />
</Form.Item>

<Modal
  title={__('刪除確認', 'power-course')}
  okText={__('確認刪除', 'power-course')}
  cancelText={__('取消', 'power-course')}
/>

// ❌ 錯誤
<Form.Item label="課程名稱" name="name">
  <Input placeholder="請輸入課程名稱" />
</Form.Item>
```

### 4. sprintf 規範（同 PHP）

```tsx
// ✅ 正確
import { sprintf, __ } from '@wordpress/i18n';

const message = sprintf(
  // translators: %s: 課程名稱
  __('課程「%s」已建立', 'power-course'),
  courseName
);

// 多參數
const message = sprintf(
  // translators: 1: 學員姓名, 2: 課程名稱
  __('%1$s 已加入課程「%2$s」', 'power-course'),
  studentName,
  courseName
);

// ❌ 錯誤：模板字串
const message = `課程「${courseName}」已建立`;

// ❌ 錯誤：直接拼接
const message = __('課程「', 'power-course') + courseName + __('」已建立', 'power-course');
```

### 5. React 元素帶在翻譯字串內

含 React 元素（如 `<a>`、`<strong>`）的翻譯，使用 `createInterpolateElement`：

```tsx
import { createInterpolateElement } from '@wordpress/element';
import { __ } from '@wordpress/i18n';

createInterpolateElement(
  __('請閱讀<a>使用條款</a>後同意', 'power-course'),
  {
    a: <a href="/terms" />,
  }
);
```

## 工具鏈使用規範

### 產生 .pot 範本

每次新增/修改翻譯字串後**必須**跑：

```bash
pnpm run i18n:pot
```

對應命令：`wp i18n make-pot . languages/power-course.pot --domain=power-course --slug=power-course --exclude=node_modules,vendor,tests,release,js/dist,languages`

產出 `languages/power-course.pot`，作為譯者起點。

### 產生 JS runtime JSON

從 `.po` 產出 React runtime 用的 `.json`：

```bash
pnpm run i18n:json
```

對應命令：`wp i18n make-json languages --no-purge --pretty-print`

產出檔名格式：`{domain}-{locale}-{handle-md5}.json`

## PR 審查驗收標準

新 PR 提交時，`wordpress-reviewer` 與 `react-reviewer` 必須檢查：

1. ✅ 新增/修改的使用者可見字串都用翻譯函式包裹（`__`, `_e`, `esc_html__` 等）
2. ✅ text domain 一律是字串字面量 `'power-course'`，無變數、無底線版本（`'power_course'` 是 bug）
3. ✅ HTML 輸出用 escape 變體（`esc_html_e` / `esc_html__` / `esc_attr_e` / `esc_attr__`）
4. ✅ 含變數的字串用 `sprintf` + placeholder，不是字串拼接或模板字串
5. ✅ 含 placeholder 的字串前有 `/* translators: ... */` 註解
6. ✅ 複數場景用 `_n()` 而非 `sprintf( __('%d ...'), $count )`
7. ✅ React 端 import 來自 `@wordpress/i18n`，不是 `i18next` 或 `react-intl`
8. ✅ Ant Design 的 label/placeholder/title/tooltip/okText 都已翻譯
9. ✅ 跑過 `pnpm run i18n:pot`，新字串已出現在 `languages/power-course.pot`

不通過則 reviewer 退回給 master agent 修正。

## 已知歷史問題

### Domain 名稱不一致（vendor，已修復）

`vendor/j7-dev/wp-plugin-trait` **0.2.19 及更早版本** 的 `PluginTrait::load_textdomain()` 會用 snake_case 呼叫 `load_plugin_textdomain('power_course', ...)`，與 plugin.php header 的 kebab-case `power-course` 不一致，導致翻譯載入到錯誤 domain。

**修復版本**：`j7-dev/wp-plugin-trait` **>= 0.2.20**（2026-04 發布）已改為使用 `self::$kebab`，正確載入 `power-course` domain。`composer.lock` 必須至少為此版本。

**歷史繞道**：`plugin.php` 曾經在 `init` priority 1 補一個手動 `load_plugin_textdomain()` 呼叫繞過此 bug，該段程式碼已於 wp-plugin-trait 升級後隨之移除（見 `issue/191` 分支 git log）。

**禁止**動 vendor 內程式碼。

### 既有硬編碼字串清理

73 個 PHP 類別檔案中只有 9 個使用翻譯函式；225+ TSX 檔案大量硬編碼。清理工作由 `wp-workflows:i18n-refactor` agent 分批協調，**不是一次性重構**。新 PR 的字串強制走 i18n，舊字串漸進式回補。
