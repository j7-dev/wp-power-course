# Power Course i18n（國際化）規範

本規範定義 Power Course 外掛在多語系支援上的強制規則。所有 PHP 與 React 程式碼必須遵守。違規視為 PR 審查不通過。

詳細 API 用法請參考 `wp-workflows:wordpress-i18n` skill。

## 核心決策

| 項目 | 決策 |
|------|------|
| Text Domain | **固定 `power-course`**（連字號，與 `plugin.php` header 一致） |
| Domain Path | `/languages` |
| **msgid 原文語言** | **英文完整句子**（WP 官方慣例）；繁體中文走 `zh_TW.po` 的 `msgstr` |
| 翻譯來源 | 單一 `.po/.mo`，PHP 與 React **共用** |
| PHP API | WordPress 原生 `__()` / `_e()` / `_x()` / `_n()` 等 |
| React API | **`@wordpress/i18n`**（已安裝），**禁止**使用 `i18next` / `react-intl` |
| 工具鏈 | `pnpm run i18n:pot`（WP-CLI `wp i18n make-pot --skip-js` 掃 PHP + `gettext-extractor` 掃 JS/TSX，`gettext-parser` 合併，詳見 `scripts/i18n-make-pot.mjs`）+ `pnpm run i18n:mo` + `pnpm run i18n:json` |
| 目標語言 | zh_TW（主要介面語言）+ en_US（fallback）+ ja（次階段） |

## 🔴 最高原則：msgid 一律是英文

WordPress i18n 的核心假設是 **msgid = 英文原文 / msgstr = 各語言翻譯**。

- ✅ `__( 'Course name', 'power-course' )`
- ❌ `__( '課程名稱', 'power-course' )`（違反 WP 慣例，非 zh_TW 站點 fallback 會顯示中文）

**理由：**

1. 英文 msgid 讓所有語系譯者（日文、西文、法文…）都看得懂原文。
2. `.po` 編輯器（Poedit, Loco Translate）與翻譯平台（GlotPress, Weblate）預設把 msgid 當英文。
3. WordPress core / WooCommerce / 主流外掛全部使用英文 msgid。
4. 當 `.mo` 載入失敗時，fallback 會直接顯示 msgid — 顯示英文遠比顯示中文對全球使用者友善。
5. `make-pot` 掃描出的字串交給外部譯者時，英文是唯一共通語言。

**translator comments 仍使用繁體中文**（因為譯者是繁中開發團隊）：

```php
sprintf(
    /* translators: %s: 課程名稱 */   ← 繁體中文，給譯者看
    __( 'Course "%s" created', 'power-course' ),   ← 英文 msgid
    $course_name
)
```

## PHP 規範

### 1. 必須使用翻譯函式

任何**使用者可見的字串**（UI 標籤、訊息、錯誤、按鈕、placeholder、admin 通知、tab 標題）必須包在翻譯函式內，禁止硬編碼。

```php
// ✅ 正確
__( 'Course name', 'power-course' )
esc_html__( 'Failed to delete course data', 'power-course' )

// ❌ 錯誤
'Course name'  // 硬編碼，無法翻譯
__( 'Course name' )  // 缺 text domain，make-pot 抓不到
__( 'Course name', 'power_course' )  // domain 錯：必須是連字號
__( '課程名稱', 'power-course' )  // msgid 必須是英文
```

### 2. 強制使用 escape 變體輸出 HTML

直接輸出到 HTML 的字串必須用 escape 版本，**禁止裸用 `_e()` 輸出**：

| 場景 | 必用函式 |
|------|---------|
| 輸出到 HTML body | `esc_html_e()` / `esc_html__()` |
| 輸出到 HTML attribute（`title="..."` / `aria-label="..."`） | `esc_attr_e()` / `esc_attr__()` |
| 輸出到 URL 屬性（`href` / `src`） | `esc_url()` 包外層 |
| 輸出到 JS context | `esc_js()` 包外層 |
| 含 HTML tag 必要的字串 | `wp_kses_post()` 包外層 |

```php
// ✅ 正確
echo esc_html__( 'Please select a course', 'power-course' );
?>
<button title="<?php esc_attr_e( 'Add to cart', 'power-course' ); ?>"></button>

// ❌ 錯誤
echo __( 'Please select a course', 'power-course' );  // XSS 風險
_e( 'Please select a course', 'power-course' );  // 同上
```

### 3. text domain 必須是字串字面量

`make-pot` 是**靜態掃描**，禁止把 text domain 放在變數內：

```php
// ✅ 正確
__( 'Course name', 'power-course' )

// ❌ 錯誤：make-pot 掃不到，永遠無法翻譯
$domain = 'power-course';
__( 'Course name', $domain )
```

### 4. sprintf + placeholder 規範

帶變數的字串**必須**用 `sprintf` + placeholder，禁止字串拼接：

```php
// ✅ 正確
sprintf(
    /* translators: %s: 課程名稱 */
    esc_html__( 'Course "%s" created', 'power-course' ),
    esc_html( $course_name )
);

// 多參數時必須用編號 placeholder（譯者可重排序）
sprintf(
    /* translators: 1: 學員姓名, 2: 課程名稱 */
    esc_html__( '%1$s joined course "%2$s"', 'power-course' ),
    esc_html( $student_name ),
    esc_html( $course_name )
);

// ❌ 錯誤：字串拼接
__( 'Course "', 'power-course' ) . $course_name . __( '" created', 'power-course' )

// ❌ 錯誤：直接代換變數
__( "Course \"{$course_name}\" created", 'power-course' )  // make-pot 掃到的字串永遠變動
```

### 5. 複數規範

數量相關的字串必須用 `_n()`：

```php
// ✅ 正確
sprintf(
    /* translators: %d: 學員人數 */
    _n(
        '%d student',
        '%d students',
        $count,
        'power-course'
    ),
    $count
);

// ❌ 錯誤
sprintf( __( '%d students', 'power-course' ), $count );
```

### 6. translator comments 慣例

含 placeholder 的字串**必須**在前一行加 `/* translators: ... */` 註解，**使用繁體中文**說明 placeholder 的意義：

```php
sprintf(
    /* translators: 1: 折扣百分比, 2: 課程名稱 */
    esc_html__( 'Get %1$d%% off course "%2$s"', 'power-course' ),
    $discount,
    esc_html( $course_name )
);
```

## React / TypeScript 規範

### 1. 必須使用 `@wordpress/i18n`

```tsx
// ✅ 正確
import { __, _x, _n, sprintf } from '@wordpress/i18n';

<Button>{__('Save course', 'power-course')}</Button>

// ❌ 錯誤：使用 i18next
import { useTranslation } from 'react-i18next';

// ❌ 錯誤：硬編碼
<Button>儲存課程</Button>

// ❌ 錯誤：中文 msgid
<Button>{__('儲存課程', 'power-course')}</Button>
```

### 2. text domain 必須與 PHP 一致

所有 JS 翻譯呼叫的第二參數**必須是 `'power-course'`**（與 PHP 共用）。`wp_set_script_translations` 已在 `inc/classes/Bootstrap.php` 接好，runtime 會自動載入對應翻譯。

### 3. Ant Design 元件文字

Ant Design 的 `label`、`placeholder`、`title`、`tooltip`、`okText`、`cancelText` 等 props 都是使用者可見字串，必須翻譯：

```tsx
// ✅ 正確
<Form.Item
  label={__('Course name', 'power-course')}
  name="name"
>
  <Input placeholder={__('Please enter course name', 'power-course')} />
</Form.Item>

<Modal
  title={__('Confirm delete', 'power-course')}
  okText={__('Confirm delete', 'power-course')}
  cancelText={__('Cancel', 'power-course')}
/>
```

### 4. sprintf 規範（同 PHP）

```tsx
// ✅ 正確
import { sprintf, __ } from '@wordpress/i18n';

const message = sprintf(
  // translators: %s: 課程名稱
  __('Course "%s" created', 'power-course'),
  courseName
);

// 多參數
const message = sprintf(
  // translators: 1: 學員姓名, 2: 課程名稱
  __('%1$s joined course "%2$s"', 'power-course'),
  studentName,
  courseName
);

// ❌ 錯誤：模板字串
const message = `課程「${courseName}」已建立`;

// ❌ 錯誤：直接拼接
const message = __('Course "', 'power-course') + courseName + __('" created', 'power-course');
```

### 5. React 元素帶在翻譯字串內

含 React 元素（如 `<a>`、`<strong>`）的翻譯，使用 `createInterpolateElement`：

```tsx
import { createInterpolateElement } from '@wordpress/element';
import { __ } from '@wordpress/i18n';

createInterpolateElement(
  __('Please read <a>terms of service</a> and agree', 'power-course'),
  {
    a: <a href="/terms" />,
  }
);
```

## 工具鏈使用規範

### 產生 .pot 範本

每次新增/修改翻譯字串後**必須**跑：

```bash
pnpm run i18n:pot
```

產出 `languages/power-course.pot`，作為譯者起點（msgid 全部是英文）。

### 產生 JS runtime JSON + 編譯 .mo

```bash
pnpm run i18n:mo     # 編譯 zh_TW.po → zh_TW.mo
pnpm run i18n:json   # 產出 React runtime 用的 JED JSON
pnpm run i18n:build  # 一次跑完 pot → mo → json
```

產出檔名格式：`{domain}-{locale}-{handle-md5}.json`

## PR 審查驗收標準

新 PR 提交時，`wordpress-reviewer` 與 `react-reviewer` 必須檢查：

1. ✅ **msgid 全部是英文完整句子**，無中日韓字元
2. ✅ 新增/修改的使用者可見字串都用翻譯函式包裹（`__`, `_e`, `esc_html__` 等）
3. ✅ text domain 一律是字串字面量 `'power-course'`，無變數、無底線版本（`'power_course'` 是 bug）
4. ✅ HTML 輸出用 escape 變體（`esc_html_e` / `esc_html__` / `esc_attr_e` / `esc_attr__`）
5. ✅ 含變數的字串用 `sprintf` + placeholder，不是字串拼接或模板字串
6. ✅ 含 placeholder 的字串前有 `/* translators: ... */` 註解（繁體中文）
7. ✅ 複數場景用 `_n()` 而非 `sprintf( __('%d ...'), $count )`
8. ✅ React 端 import 來自 `@wordpress/i18n`，不是 `i18next` 或 `react-intl`
9. ✅ Ant Design 的 label/placeholder/title/tooltip/okText 都已翻譯
10. ✅ 若新詞彙不在術語表，已先 PR 更新術語表
11. ✅ 跑過 `pnpm run i18n:pot`，新字串已出現在 `languages/power-course.pot`
12. ✅ `languages/power-course-zh_TW.po` 已補上對應 `msgstr` 繁體中文翻譯

不通過則 reviewer 退回給 master agent 修正。

## 已知歷史問題

### Domain 名稱不一致（vendor，已修復）

`vendor/j7-dev/wp-plugin-trait` **0.2.19 及更早版本** 的 `PluginTrait::load_textdomain()` 會用 snake_case 呼叫 `load_plugin_textdomain('power_course', ...)`，與 plugin.php header 的 kebab-case `power-course` 不一致，導致翻譯載入到錯誤 domain。

**修復版本**：`j7-dev/wp-plugin-trait` **>= 0.2.20**（2026-04 發布）已改為使用 `self::$kebab`，正確載入 `power-course` domain。`composer.lock` 必須至少為此版本。

**禁止**動 vendor 內程式碼。

### WooCommerce text domain 借用（已豁免）

`inc/templates/components/review/comment_form.php` 的 `title_reply` 與 `label_submit` 欄位使用 `'woocommerce'` text domain，刻意借用 WooCommerce 現成翻譯（"Add a review"、"Submit"），以共享各語言的 WC 翻譯資源。

**豁免理由**：減少翻譯維護成本，且 WooCommerce 的翻譯品質在主流語言中已被驗證。

**限制範圍**：僅限 `comment_form.php` 這個 WC 整合介面的 `comment_form()` 參數；其餘所有模板一律使用 `'power-course'` domain。

### 歷史中文 msgid 遷移（2026-04 完成）

2026-04 之前專案曾錯誤地將繁體中文當 msgid 使用（`__('課程名稱', 'power-course')`）。已於 `issue/191` 分支一次性重構為英文 msgid，並同步產出 `languages/power-course-zh_TW.po` 保證繁中介面不受影響。重構後**絕對禁止**再引入中文 msgid。
